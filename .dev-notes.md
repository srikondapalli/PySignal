### Hook usage inventory (exact call sites to shim)
- curriculum.list.useQuery(undefined, opts) → LessonMeta[]
- curriculum.lesson.useQuery({id}, opts) → Lesson minus correctIndex (quiz: QuizQuestionPublic[]), plus error handling
- progress.summary.useQuery(undefined?, opts) → { progress: {lessonId,status,...}[], activityDates: string[], bestScores: Record<id,{score,total,attempts}> }
- progress.setStatus.useMutation → input {lessonId,status,localDate}; onSuccess(_d, vars)
- progress.ping.useMutation → input {localDate}
- quiz.submit.useMutation → input {lessonId, answers, localDate} → {score,total,results:[{correct,correctIndex,explanation}]}
- quiz.history.useQuery({lessonId}) → rows w/ id, score, total, createdAt (newest first)
- notes.get.useQuery({lessonId}) → row {content,...} | null; notes.list.useQuery() → rows w/ lessonId, content, updatedAt; notes.save.useMutation({lessonId,content})
- trpc.useUtils() → utils.X.invalidate(...) calls: quiz.history, progress.summary, notes.get, notes.list, auth.me.setData
- useAuth() → { user, loading, error, isAuthenticated, logout }

### Chosen implementation (minimal page edits)
Create client/src/lib/api.ts exporting an `api` object mimicking the trpc client surface (api.curriculum.list.useQuery etc.) plus `useApiUtils()` and `useCourseAuth()`.
In server mode these delegate to trpc/useAuth; in static mode they are implemented over localStorage + bundled JSON.
Pages swap `import { trpc } from "@/lib/trpc"` → `import { api as trpc, useApiUtils } from "@/lib/api"` and `trpc.useUtils()` → `useApiUtils()`; useAuth import swaps to useCourseAuth. This keeps all call-sites unchanged.
Static quiz grading needs correctIndex: bundle full lesson JSON (fine; public content) via import.meta.glob eager.
